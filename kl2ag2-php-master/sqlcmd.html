Operacje na tabelach
1.Tworzenie tabeliCREATE TABLE [IF NOT EXIST] nazwa_tabeli (nazwa_pola właściwości_pola) ENGINE = MYISAM;
 np.: CREATE dane (id SMALLINT UNSIGNED NOT NULL AUTO_INCREMENT PRIMARY KEY, imie VARCHAR(20) NOT NULL, nazwisko VARCHAR(20) NOT NULL, uwagi TEXT NOT NULL) ENGINE = MYISAM;
 Uwaga: informacje o typach danych można uzyskać po zalogowaniu się do bazy (patrz wyżej) i nast. poleceń:
  mysql> help Data Types – wyświetla nazwy typów
mysql> help nazwa_typu – wyświewtla informacje o podanym typie danych, np. help INT
Typy pól SMALINT – pole liczbowe (-32768 do 32768 lub 0 do 65535)
INT – pole liczbowe (-2147483648 do 2147483647 lub 0 do 4294967295)
FLOAT – pole liczbowe (-3.402823466E+38 do -1.754494351E-38 lub 0 i -1.754494351E-38 do 3.402823466E+38)
DATE – data w formacie RRRR-MM-DD
DATETIME – data i czas w formacie RRRR-MM-DD GG:MM:SS
TIMESTAMP – data i czas liczone w sekundach od początku epoki UNIX-a, czyli od 1970-01-01 00:00:00
CHAR – pole znakowe o stałej długości (1 do 255 bajtów)
VARCHAR – pole znakowe o zmiennej długości (1 do 255 bajtów)
TINYTEXT – tekst nieprzekraczający 255 bajtów
TEXT – tekst nieprzekraczający 65 535 bajtów
ENUM – pole zawierające jedną z wartości wymienionych na liście wyliczeniowej definicji pola
SET – pole zawierające dowolną liczbę z wartości wymienionych na liście wyliczeniowej definicji pola
Modyfikatory typów pól [NOT] NULL – pole (nie) może zawierać wartości null (nic)
AUTO_INCREMENT – wartość pola będzie nadawana automatycznie
PRIMARY_KEY – pole zdefiniowane jako klucz główny, dla pola tworzony jest indeks
UNSIGNED – oznacza "bez znaku" i zmienia zakres zmiennych typu liczbowego na dodatni
BINARY – pole zawiera wartości binarne (np. obrazki, dźwięki)
DEFAULT wartość – określa wartość domyślną pola
ENGINE = MYISAM – określa typ silnika bazy danych

Operacje na danych
1.Dodawanie danych do tabeliINSERT INTO nazwa_tabeli (nazwa_pola1,[nazwa_pola2]...) VALUES ('wartość1',['wartość2']...);
 np.: INSERT INTO dane (id,imie,nazwisko,uwagi,data) VALUES (NULL,'Jan','Kowalski','Fajny gość!','1982-10-26');
2.KwerendySELECT [*]|[nazwa_pola1,nazwa_pola2...] FROM nazwa_tabeli
 np.: SELECT * FROM dane; lub SELECT imie from DANE;
* oznacza wszystkie pola
Modyfikatory kwerend LIMIT nr_rekordu, ilość_rekordów – kwerenda zwróci ilość_rekordów poczynając od nr_rekordu
np.: SELECT imie,nazwisko FROM dane LIMIT 2,5;
ORDER BY 'nazwa_pola' ASC|DESC – zwróceone rekordy zostaną posrtowane wg 'nazwy_pola' roscnąco(ASC) lub malejąco(DESC)
np.: SELECT * FROM dane ORDER BY 'nazwisko' ASC;
WHERE warunek – zwraca rekordy spełniające warunek
np.: SELECT imie FROM dane WHERE imie='Ala';
SELECT nazwisko FROM dane WHERE data_ur>1990-01-10;
Operatory warunków Określając warunek w klauzuli WHERE można używać operatorów: = (jest równe), > (jest większe), >= (większe lub równe), < (mniejsze), <= (mniejsze lub równe), != (nie jest równe), [NOT] LIKE (nie jest/jest podobne), IS [NOT] NULL (jest/nie jest puste), [NOT] REGEXP (nie jest/jest równe wyrażeniu)
np.: SELECT * FROM dane WHERE 'imie' LIKE '%a%'; (znak % zastępuje dowolną część ciągu)
Dostępne są również operatory logiczne AND (koniunkcja) i OR (alternatywa) – wykorzystywane do grupowania warunków – pamiętać należy o właściwym używaniu nawiasów precyzujących wynik wyrażenia.
np.: SELECT * FROM dane WHERE (imie LIKE '%a%' AND nazwisko='Kowalski')
3.Aktualizacja danychUPDATE nazwa_tabeli SET nazwa_pola VALUES ('wartość') WHERE warunek;
 np.: UPDATE dane SET imie VALUES ('Adam') WHERE nazwisko='Kowalski';
4.Usuwanie danychDELETE FROM nazwa_tabeli WHERE warunek;
 np.: DELETE FROM dane WHERE nazwisko='Kowalski';
